# 뭐갖고싶어 Visual System Design

## Purpose

`뭐갖고싶어`의 디자인은 공개 공유 경험과 관리자 운영 경험을 분리합니다.

공개 페이지는 친구에게 보내는 생일 카드처럼 보여야 합니다. 귀엽고 가볍고, 링크를 받은 사람이 바로 선물 목록을 훑고 마음을 남기고 싶어야 합니다.

관리자 화면은 반복 입력과 상태 관리를 위한 운영 UI입니다. 공개 페이지의 장식성을 그대로 가져오지 않고, 조용하고 읽기 쉬운 화면으로 유지합니다.

## References

- 기존 레퍼런스 repo: `song-chaeyoung/birthday-wishlist`
- 레퍼런스의 공개 페이지는 `pixel-card`, `dot-bg`, `sticker-label`, 두꺼운 테두리, 강한 그림자, 파스텔 배경을 사용합니다.
- 레퍼런스 README는 방문자가 선물 목록을 확인하고 비공개 축하 메시지를 남기는 흐름을 핵심 기능으로 설명합니다.
- 현재 프로젝트 문서는 기본 공개 테마를 `pixel_y2k`로 정의하고, 테마 적용 범위를 `/b/[slug]`, OG 이미지, 공유 미리보기로 제한합니다.

## Core Decision

추천 방향은 `Public Pixel / Admin Calm`입니다.

공개 화면:

- `pixel_y2k`를 기본 브랜드 인상으로 사용합니다.
- 두꺼운 테두리, 오프셋 그림자, 파스텔 면, 스티커 라벨, 작은 픽셀 장식을 씁니다.
- 친구끼리 공유하는 생일 위시리스트처럼 장난스럽게 보이되, 선물명, 금액, 진행률, 상품 링크는 빠르게 읽혀야 합니다.

관리자 화면:

- 테마를 적용하지 않습니다.
- 배경은 밝은 뉴트럴, 입력/리스트는 낮은 대비의 선과 작은 그림자로 정리합니다.
- 선물 등록, 수정, 삭제, 메시지 확인처럼 반복 작업이 많은 화면이므로 장식보다 밀도와 안정감을 우선합니다.

## Visual Tokens

공개 `pixel_y2k` 기본값:

| Token | Value | Usage |
| --- | --- | --- |
| Ink | `#171717` | 주 텍스트, 두꺼운 테두리 |
| Deep Purple | `#4c1d95` | 공개 페이지 헤드라인, 강조 잉크 |
| Teal | `#0f766e` | 진행률, 성공 상태, 보조 CTA |
| Orange | `#f97316` | 선물 아이콘, 주요 포인트 |
| Yellow | `#fef3c7` | 히어로/스티커 배경 |
| Mint | `#ccfbf1` | 카드 이미지 fallback, 보조 면 |
| Blush | `#ffe4e6` | 공개 페이지 배경 포인트 |
| Paper | `#fffdf7` | 카드 내부 배경 |
| Neutral Line | `#d1d5db` | 관리자 UI 경계 |

형태:

- 공개 카드 border: `2px solid #171717`
- 공개 카드 shadow: `6px 6px 0 #111827`, 히어로는 `8px 8px 0 #111827`
- 관리자 카드 border: `1px solid #d1d5db`
- radius: 버튼과 입력은 `6px` 이하, 큰 공개 카드는 직각 또는 `4px` 이하
- gradient orb, bokeh, 추상 SVG 배경은 쓰지 않습니다.

타이포:

- 기본은 시스템 sans를 사용합니다.
- 공개 페이지 헤드라인은 `font-black`, 본문은 14-16px 중심으로 갑니다.
- 관리자 화면은 `font-semibold` 이하를 주로 쓰고, 헤드라인 크기를 과하게 키우지 않습니다.
- viewport width 기반 글자 크기는 쓰지 않습니다.

## Page Rules

### Home

첫 화면은 제품명과 실제 공개 페이지 미리보기가 보여야 합니다.

- H1: `뭐갖고싶어`
- 보조 문구: 생일자가 받고 싶은 선물을 링크 하나로 공유한다는 문장
- 우측/하단 미리보기: 실제 `/b/[slug]` 공개 페이지 카드 구조를 축약한 형태
- CTA: 비로그인은 `내 위시리스트 만들기`, 로그인 후 온보딩 미완료는 `위시리스트 만들기 계속하기`, 온보딩 완료는 `내 위시리스트 관리하기`

### Public Wishlist `/b/[slug]`

공개 페이지는 브랜드 감성을 가장 강하게 보여주는 화면입니다.

- 상단은 생일 카드 포스터처럼 구성합니다.
- 타이틀은 `{ownerName}님이 갖고 싶은 것들` 또는 wishlist title을 크게 보여줍니다.
- 요약 박스는 `선물`, `모인 마음`처럼 감성적인 라벨을 씁니다.
- 선물 카드는 이미지 영역, 상태 스티커, 선물명, 설명, 진행률, 상품 링크를 한 번에 읽게 합니다.
- 이미지가 없으면 `GIFT` 텍스트만 두지 않고, 픽셀 선물 상자 fallback을 둡니다.
- `hidden` 선물은 노출하지 않습니다.

### Public Empty / 404

빈 상태와 404도 공개 테마를 따릅니다.

- 빈 상태: `아직 공개된 선물이 없어요`
- 404: `위시리스트를 찾을 수 없어요`
- CTA는 `처음으로` 또는 `내 위시리스트 만들기` 하나만 둡니다.

### Admin

관리자 화면은 기능 중심으로 유지합니다.

- 공개 테마 장식은 넣지 않습니다.
- 상단 nav, 입력 폼, 리스트 카드의 간격과 버튼 스타일만 정리합니다.
- 상태 라벨은 작고 명확하게 유지합니다.
- 공개 페이지 미리보기 링크만 브랜드 컬러로 강조합니다.

## Copy Tone

카피는 짧고 친구 같은 말투를 씁니다.

사용할 문장:

- `생일 축하해. 뭐 갖고 싶어?`
- `받고 싶은 선물을 링크 하나로 모아 공유해요.`
- `{name}님이 갖고 싶은 것들`
- `마음 모으는 중`
- `상품 보기`
- `내 위시리스트 만들기`
- `위시리스트 만들기 계속하기`
- `내 위시리스트 관리하기`

피할 문장:

- 지나치게 SaaS처럼 보이는 `관리 솔루션`, `운영 대시보드`
- 공개 페이지에서 딱딱한 `목표 금액`, `누적 금액`만 단독으로 쓰는 표현
- 기능 설명을 화면에 길게 늘어놓는 문장

## Implementation Order

1. `app/globals.css`에 공개/관리 공통 토큰을 CSS 변수로 정리합니다.
2. 깨진 한글 카피를 홈, 공개 페이지, 어드민에서 먼저 복구합니다.
3. `/b/[slug]`에 `pixel_y2k` 테마를 실제로 적용합니다.
4. 홈 화면 미리보기를 공개 페이지 카드와 같은 visual language로 맞춥니다.
5. 어드민은 장식 추가가 아니라 간격, 라벨, 입력 상태, 버튼 위계를 정리합니다.
6. 이후 메시지 폼과 OG 이미지는 이 기준을 그대로 따릅니다.

## Non Goals

- 여러 테마를 지금 구현하지 않습니다.
- 애니메이션, 마스코트, 복잡한 배경 이미지를 먼저 만들지 않습니다.
- 관리자 화면을 공개 페이지처럼 꾸미지 않습니다.
- 디자인 시스템 패키지나 별도 UI 라이브러리를 추가하지 않습니다.
