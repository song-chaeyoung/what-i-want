Admin Wishes Mobile References

현재 선물 관리 화면은 모바일에서 상단 여백, 큰 필터 카드, 긴 입력 폼이 한 번에 노출되어 실제 관리 작업보다 양식이 먼저 보입니다. 이 보고서는 Lazyweb에 저장되어 있던 Stripe Dashboard와 Linear 레퍼런스를 선물 관리 화면에 맞게 다시 정리한 것입니다.

TL;DR 선물 관리 화면은 "폼을 먼저 보여주는 페이지"가 아니라 "선물 상태를 빠르게 스캔하고 필요한 때 추가하는 페이지"로 바꿔야 합니다. 모바일 기준으로는 Linear식 조밀한 상태 탭 + 리스트를 기본 화면으로 두고, 선물 추가 폼은 Stripe식 별도 패널/하단 CTA 성격으로 낮추는 편이 맞습니다.

Recommended Shape

1. 상단은 짧은 컨텍스트와 핵심 CTA만 둡니다

지금처럼 공개 링크, 제목, 버튼, 필터가 각각 큰 박스로 쌓이면 첫 화면이 무거워집니다. Stripe Dashboard의 모바일 홈은 지표를 작은 스트립으로 압축하고, 주요 작업은 플로팅/상단 액션으로 둡니다. 

┌──────────────────────────┐
│ /b/w-a0cb...        보기 │
│ 선물 관리               +│
│ 0개 · 모인 금액 ₩0       │
└──────────────────────────┘

2. 상태 필터는 큰 버튼 묶음이 아니라 탭으로 낮춥니다

Linear의 이슈 화면은 상태/범위를 탭으로 처리하고, 리스트가 바로 이어집니다. 이 화면에는 복잡한 검색보다 "전체 / 모으는 중 / 일시중지 / 완료 / 숨김"을 얇은 segmented tab으로 유지하는 정도가 적절합니다. 

[전체] [모으는 중] [일시중지] [완료] [숨김]
────────────────────────────
○ 맥북 파우치         모으는 중
  ₩12,000 / ₩45,000       27%
────────────────────────────
○ 케이크               완료
  ₩35,000 / ₩35,000      100%

3. 선물 추가 폼은 기본 노출을 줄이고, 필요할 때 펼칩니다

현재 화면의 가장 큰 문제는 빈 목록에서도 긴 추가 폼이 화면을 대부분 차지한다는 점입니다. Stripe의 Add customer 패턴처럼 생성 폼은 명확한 제목, 선택 필드 라벨, 하단 CTA를 가진 별도 패널로 다루는 편이 낫습니다. 

기본 화면:
[+ 선물 추가]

추가 모드:
┌ 선물 추가 ───────────────┐
│ 선물 이름                 │
│ 목표 금액                 │
│ 상품 링크        선택     │
│ 이미지 링크      선택     │
│ [선물 저장]               │
└──────────────────────────┘

4. 선물 항목은 카드보다 리스트 행에 가깝게 만듭니다

Stripe 고객 목록처럼 이름, 보조 정보, 이동/수정 액션만 한 줄에 정리하면 모바일에서 관리 화면답게 빨라집니다. 선물 이미지는 작은 썸네일로 충분하고, 진행률은 얇은 바와 금액 텍스트로 처리하는 편이 좋습니다.

Implementation Direction

  • 모바일 첫 화면 높이를 줄입니다: 헤더, 공개 링크 CTA, 상태 탭을 한 화면 안에 압축합니다.
  • 선물 추가는 접힘/별도 섹션으로 이동합니다: 빈 상태일 때만 안내와 CTA를 강하게 둡니다.
  • 선물 목록은 카드 그리드가 아니라 리스트 행으로 설계합니다: 썸네일, 이름, 상태 뱃지, 진행률, 수정 액션.
  • 데스크톱은 같은 데이터를 2열로 늘리지 말고, 왼쪽 리스트 + 오른쪽 편집 패널 또는 테이블형 관리 뷰로 갑니다.
  • 공개 페이지 테마 토큰은 사용하지 않습니다. 관리자 화면은 계속 calm admin 톤이어야 합니다.
